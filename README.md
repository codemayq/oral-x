## Introduction
This project uses the Qwen3-0.6B-Embedding model to encode text features, ResNet-101 to encode conventional image features, and Virchow2 to encode pathology slide features for multi-label classification of medical cases. The overall model is a multimodal architecture based on NLP and ViT, where the weights of Virchow2 and Qwen3-Embedding are frozen and do not participate in weight updates during training. The parts that are updated include: ResNet, the MIL model GatedAttentionMIL, and the UnifiedMultimodalModel that handles projection, modality alignment, and fusion, along with its built-in classification head.



## Parameters

### Configuration File Parameter Descriptions

`RAW_DATA_ROOT:` Root directory of the dataset, e.g. `/media/codingma/LLM/lcx/Medical_Info_Classification/datasets/train`

`DATA_INDEX_PATH:` Directory where the dataset index file is stored. This file can be auto-generated by the `MIL/prepare_dataset.py` script to record how many data files each sample has. Simply run `python prepare_dataset.py` to generate the index file at the path specified by `DATA_INDEX_PATH`.

`OVERWRITE_SWI_FEATURES:` Whether to regenerate WSI features when running the prepare_dataset script. If True, Virchow2 will re-extract features for each WSI file. If False, feature extraction will be skipped for WSIs that already have extracted feature files.

`VIRCHOW2_MODEL_ID:` Virchow2 model weight path or Hugging Face ID. This model requires access permission. It should be passed in one of the following two forms: 1. "hf-hub:paige-ai/Virchow2"; 2. "local-dir:/media/codingma/LLM/lcx/Virchow2" — 1 is the online path, 2 is the local path.

`QWEN_MODEL_PATH:` Qwen3-Embedding model weight path or Hugging Face URL

`CLASS_MAP:` Mapping from class ID to class name

`CHECKPOINT_DIR:` Training checkpoint save path

`NUM_CLASSES:` Number of output classes for the classification head

## Model

The model consists of the following components. See `MIL/src/mil/model.py` for detailed structure:

1. Text backbone for text feature extraction: Qwen3-Embedding model.
2. Image backbone for conventional image feature extraction: the backbone is ResNet with the last two layers (AvgPool, FC) removed, retaining only the feature layers.
3. Multi-Instance Learning model for pathology slide feature processing: Masked Gated Attention MIL (GatedAttentionMIL).
4. Transformer-encoder for cross-modal fusion: Cross-Modal Fusion model.
5. A `fusion_cls_token` for aggregating all the above feature classification information, and a classifier for outputting the final classification results.


## Dataset
The current dataset is stored at `/media/codingma/LLM/lcx/Medical_Info_Classification/datasets`. Sample files under the `train` and `test` directories at this path are all symbolic links. Deleting them does not affect the original files. If re-splitting is needed, they can be cleared at any time.

Before starting training, prepare the dataset according to the following steps:

1. **Extract content from doc files to generate txt text files**  
   This step can be completed by the `doc2txt` function provided in `MIL/prepare_dataset.py`.

2. **Offline feature extraction for WSI data**  
   Through the `generate_wsi_features` function provided in `MIL/prepare_dataset.py`, features can be extracted offline for all WSI data under the `RAW_DATA_ROOT` path. This function accepts an `overwrite` bool parameter: when True, feature files are regenerated every run; when False, it skips WSIs that already have extracted features and only extracts features for those without. The generated features are stored as a large 2D Tensor in a `.pt` file with the same name as the `.svs` file, in the same directory.

3. **Split training and test datasets**  
   Through the `prepare_dataset` function provided in `MIL/prepare_dataset.py`, the dataset can be split into training and test sets via symbolic links, randomly partitioned according to the provided `train_ratio`. The default split ratio in the current code is 0.5. Note: when the number of samples for a certain class in the original data is less than 50, the `train_ratio` will be forcibly increased to 0.8.

4. **Generate an index file for the training dataset**  
   The index file is used to obtain the paths of individual files in each dataset. It is a CSV table and can be generated through the `generate_index_file` function provided in `MIL/prepare_dataset.py`. The index file is saved at the path specified by `DATA_INDEX_PATH` in `config.py`.

After completing the above data preparation, model training can begin. See below for the training workflow.


## Training

Before starting training, prepare the data index file as described. If re-splitting of the test set is needed, this can be done before generating the index file using the script provided in `Medical_Info_Classification/MIL/src/mil/utils.py`. After configuring the relevant split parameters, delete the existing symbolic links or specify a new save path, and re-run.

```shell

cd /media/codingma/LLM/lcx/Medical_Info_Classification/MIL

# Install
pip install -e .


# To start training:

python train.py

```

Two checkpoint files are saved: one is the model weights with the best validation accuracy at each epoch (`best_val.pth`), the other is the weights at the end of training (`last.pth`). Use `best_val.pth` as the primary checkpoint.


## Inference

`inference.py` provides the inference script, with the entry point `evaluate_test_set`. This function accepts a `test_root` parameter specifying the path to the test dataset, and three optional bool parameters (`use_txt`, `use_img`, `use_svs`), all defaulting to True. Setting them to False disables text, photo, and slide data respectively. It also accepts an `enable_fallback` bool parameter, which controls whether to guarantee at least one label is selected in multi-label classification. When True, even if all output probabilities are below the threshold, the label with the highest probability will still be selected. When False, if all label probabilities are below the threshold, the sample's classification label will be empty.

The output results include the following:

1. Number of samples used for inference and sample distribution

2. Test Accuracy

3. Classification Report, including precision, recall, f1-score, macro avg, weighted avg, and other metrics

4. Confusion Matrix and ROC curve

cd /media/codingma/LLM/lcx/Medical_Info_Classification/MIL
